# UI 검증 및 더미 항목 크래시 수정

인수인계 문서의 미확인 항목("버튼 글씨 잘림이 고쳐졌는지 검증 필요")을 확인하고,
그 과정에서 발견한 크래시를 고친 기록이다.

코드 변경은 2개 파일 6줄이다. 레이아웃은 손대지 않았다.


## 1. 검증 방법

기존에는 화면을 볼 수 없어 검증이 막혀 있었다. 이번에는 Qt의 오프스크린
플랫폼(`QT_QPA_PLATFORM=offscreen`)으로 UI를 실제로 기동하고, `QWidget.grab()`으로
각 화면을 PNG로 렌더링해 눈으로 확인했다.

검증 환경은 PyQt6 / Qt 6.11.0이다. 렌더링만 오프스크린이고, 위젯 배치와 크기
계산은 실제 실행과 동일한 경로를 탄다.

확인한 화면은 다음 4개다.

| 화면 | 창 크기 |
|---|---|
| 등록 화면 - 행동 분석 탭 | 1120x800 (기본) |
| 등록 화면 - 화면 분석 탭 | 1120x800 |
| QA 결과 화면 - 행동 분석 탭 | 1120x800 |
| 등록 화면 - 행동 분석 탭 | 900x680 (최소) |


## 2. 검증 결과: 레이아웃은 정상이다

버튼 글씨 잘림은 없다. 최소 크기 900x680 에서도 가장 긴 라벨인
"봇으로 생성 (시연용 · 실제 배포 시엔 회사 제공)"이 온전히 표시되고, 버튼이
그룹박스 테두리를 뚫는 현상도 없다.

`stackedWidget`을 절대좌표에서 레이아웃 기반으로 바꾼 조치(`centralwidget`에
`QVBoxLayout` 부여)가 의도대로 동작한다. 원래 561px로 고정돼 위젯이 눌리던
문제는 해결됐다.

즉 인수인계 문서의 미확인 항목은 해소됐다. 추가 조정은 필요 없다.


## 3. 발견한 크래시: .ui에 박힌 더미 항목

팀원의 `qa_first.ui`에는 `errorReportHistory` 위젯에 디자인용 더미 항목 4개가
하드코딩돼 있다.

```
이건 오류보고서 히스토리
새 항목1
새 항목2
새 항목3
```

이 항목들은 QA 결과 화면에 그대로 뜨고, 클릭하면 죽는다.

```
KeyError: '이건 오류보고서 히스토리'
```

원인은 `logic.show_error_detail`이 `dummy_ai_reports[item.text()]`로 상세를
찾는데, 더미 항목은 그 dict에 키가 없기 때문이다.

### 왜 실제로 밟히는 경로인가

`display_report`가 목록을 지우므로 QA 실행이 끝난 뒤에는 더미가 사라진다.
그래서 문제가 없어 보이지만, `main._on_run`은 워커를 시작하기 **전에** 결과
화면으로 전환한다.

```
self.stackedWidget.setCurrentWidget(self.qa_window)   # 먼저 전환
w = logic.QARunWorker(game, mode); w.start()          # 그 다음 봇 실행 시작
```

따라서 봇이 도는 수십 초 동안 사용자에게 더미 4개가 보이고 클릭할 수 있다.
시연 중에 밟기 쉬운 자리다. `actionerror_plus` 팝업 경로도 결과 화면으로
전환하므로 동일하다.


## 4. 수정

`.ui` 무변경 원칙을 지켜야 하므로 `.ui` 파일은 고치지 않고 코드에서 처리했다.

### main.py - `_build_result_tabs`

결과 화면을 조립할 때 목록을 비운다. 이것이 실제 수정이다.

```python
self.errorReportHistory.clear()
```

### logic.py - `show_error_detail`

상세가 없는 항목이 클릭돼도 죽지 않도록 방어한다. 안전망이다.

```python
self.errorReport.setText(
    dummy_ai_reports.get(item.text(), "(이 항목의 상세 내용이 없습니다)"))
```


## 5. E2E 검증

수정 후 UI를 통해 실제 파이프라인을 끝까지 돌렸다. 미리 만든 리포트를 읽는 것이
아니라 실제 봇이 플레이한다.

| 단계 | 결과 |
|---|---|
| 결과 화면 최초 목록 | 비어 있음 (더미 사라짐) |
| 정상 데이터 생성 | 40건 |
| 기준 모델 학습 및 저장 | `model_exists` 참 |
| QA 실행 (시연 모드) | 12판, 규칙 위반 3판, 이상 9판 |
| 리포트 표시 | 트리아지 9건, 최상단 `[규칙] 판 #9 · stuck` |
| 항목 클릭 후 상세 | 정상 표시 |

규칙 위반 판이 최상단에 오고 그 아래가 이상 점수 내림차순이라는 리포트 정렬
원칙이 UI에서도 그대로 보인다.


## 6. 안 고친 것

팀원 원본 흐름의 잔재다. 동작에 문제가 없고, 팀원 연동 때 다시 볼 자리라
이번에는 기록만 남긴다.

| 항목 | 상태 |
|---|---|
| `start_window` 페이지 | 빈 껍데기. 위젯 4개(`gameFileOpen`, `txtFileOpen`, `gameFileRoute`, `txtFileRoute`)는 비전 탭으로 옮겨졌고 `qaStart`만 고아로 남아 있다. `stackedWidget`의 0번 페이지지만 표시되지 않는다. |
| `qaStart` 버튼 | 아무 데도 연결되지 않음 |
| `logic.go_to_qa_window()` | 호출하는 곳 없음. `qaStart`의 원래 핸들러였다. |

원래 팀원 설계는 `start_window`에서 `qaStart`를 눌러 `qa_window`로 가는 흐름이었고,
우리가 그 자리를 등록 화면(`register_page`)으로 대체하면서 남은 것이다.
