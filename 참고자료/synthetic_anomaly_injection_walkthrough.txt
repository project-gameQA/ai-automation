SYNTHETIC ANOMALY INJECTION - STEP-BY-STEP WALKTHROUGH
===========================================================


STEP 1: PICK YOUR SCRIPTING PATH (do this first, before writing any injection logic)
-----------------------------------------------------------------------------------
Unreal gives you two ways to script the editor: Editor Utility Blueprints (visual scripting)
or Python via the Editor Scripting Utilities plugin. For this project, use Python - you'll
want to loop over hundreds of actors programmatically, generate structured JSON metadata
alongside screenshots, and eventually call this from a batch/CI script outside the editor.
Blueprint makes all of that clunkier.

Setup:
1. Enable "Editor Scripting Utilities" and "Python Editor Script Plugin" in Edit -> Plugins.
2. Confirm you can run a one-liner from the Python console in-editor
   (Window -> Developer Tools -> Python Console):
       import unreal; print(unreal.EditorLevelLibrary.get_all_level_actors())
3. If that returns actors, you're set up correctly.


STEP 2: GET 2-3 FREE SAMPLE PROJECTS TO INJECT BUGS INTO
---------------------------------------------------------
Don't build your own test scene first - you want diversity of art style from day one, since
that's what makes the classifier generalize (defect signatures like checkerboard textures or
floating silhouettes should look structurally similar across different art styles). Pull down:
  - Unreal's own Lyra sample or Third Person Template
  - 1-2 free Marketplace/Fab samples with different art styles (stylized + realistic if you
    can find both)

This also mirrors the "one-time setup per game" pattern from TITAN - you're proving the
injection script works across projects, not just one.


STEP 3: WRITE ONE INJECTION TYPE FIRST, END-TO-END, BEFORE GENERALIZING
-------------------------------------------------------------------------
Get the full loop working for a single bug class first - start with missing/null material
since it's the simplest to detect visually (checkerboard is unmistakable) and cheapest to
script.

Example script:

    import unreal
    import json
    import os

    def inject_missing_material(actor, output_dir, sample_id):
        mesh_comp = actor.get_component_by_class(unreal.StaticMeshComponent)
        if not mesh_comp:
            return None

        # Save original materials so we can revert
        original_materials = mesh_comp.get_editor_property("override_materials")

        # Inject: null out material slot 0
        num_slots = mesh_comp.get_num_materials()
        mesh_comp.set_material(0, None)

        # Capture screenshot
        screenshot_path = os.path.join(output_dir, f"{sample_id}.png")
        unreal.AutomationLibrary.take_high_res_screenshot(
            1280, 720, screenshot_path
        )

        # Write labeled metadata
        metadata = {
            "sample_id": sample_id,
            "bug_class": "missing_material",
            "actor_name": actor.get_actor_label(),
            "image_path": screenshot_path,
        }
        with open(os.path.join(output_dir, f"{sample_id}.json"), "w") as f:
            json.dump(metadata, f)

        # Revert so the level is clean for the next injection
        mesh_comp.set_editor_property("override_materials", original_materials)

        return metadata

Run this against a single actor manually first. Check the screenshot actually shows the
checkerboard. This one-injection-working checkpoint matters more than it sounds - it
validates your revert logic, your screenshot capture, and your metadata format all at once,
before you scale up.


STEP 4: BUILD THE LOOP THAT APPLIES IT ACROSS EVERY MESH ACTOR IN A LEVEL
----------------------------------------------------------------------------
Once step 3 works on one actor, wrap it in a loop:

    def generate_missing_material_dataset(output_dir, samples_per_actor=1):
        all_actors = unreal.EditorLevelLibrary.get_all_level_actors()
        mesh_actors = [a for a in all_actors if a.get_component_by_class(unreal.StaticMeshComponent)]

        results = []
        for i, actor in enumerate(mesh_actors):
            sample_id = f"missing_material_{i:04d}"
            meta = inject_missing_material(actor, output_dir, sample_id)
            if meta:
                results.append(meta)

        return results

This gets you your first real dataset slice: N labeled images, one bug class, zero manual
annotation.


STEP 5: ADD THE "NEGATIVE" (NORMAL) SAMPLES IN THE SAME PASS
-----------------------------------------------------------------
Before moving to the other 4 bug classes - capture clean screenshots of the same actors,
unmodified, first. You need this for two reasons:
  (a) your synthetic classifier needs negative examples too, not just positives, and
  (b) this is essentially free right now since you already have the actor loop built.


STEP 6: NOW GENERALIZE TO THE OTHER 4 INJECTION TYPES
----------------------------------------------------------
Once the pattern above is proven, the other four are mostly copy-paste with different
set/revert logic:

  Bug class            | Injection mechanic
  ---------------------|------------------------------------------------------------------
  Clipping             | Disable SetCollisionEnabled(NoCollision) on the mesh component
  Animation freeze     | Pause the Anim Blueprint at a mid-blend frame (set bPauseAnims or
                        | manually set a fixed pose)
  Floating/out-of-bounds| Offset actor Z location up or below floor via set_actor_location
  UI overlap           | Slightly harder - needs a UMG widget manipulation pass rather than
                        | an actor loop, so tackle this last

Do UI overlap last since it's structurally different (widget tree, not actor/component), and
the other four all share your existing actor-loop scaffolding.


STEP 7: ORGANIZE THE OUTPUT INTO A REAL DATASET STRUCTURE
----------------------------------------------------------------
Once you're generating across multiple bug classes and multiple sample projects, structure
matters:

    dataset/
      missing_material/
        project_lyra/    (images + json)
        project_samplex/
      clipping/
        project_lyra/
        ...
      normal/
        project_lyra/
        ...

Keeping project name in the path lets you later split train/val by project (so your
classifier is validated on an art style it never saw during training - the actual test of
generalization).


STEP 8: ONE THING TO DECIDE BEFORE YOU SCALE UP
-----------------------------------------------------
Do you want to run this injection script manually per level, or wire it into a commandlet
that can run headlessly across many levels/projects in a batch (closer to a real CI
pipeline)? Manual is fine to prove the concept works this week; a commandlet is worth doing
once you're confident in the injection logic, since manually opening levels one by one gets
old fast once you have 5 bug classes x 3 projects x many actors.
